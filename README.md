And probably emojis or anything else that isn't just plain English text.
Sorry I don't think I can fix it since it's a python issue.
Thankfully you don't really need a script to fix it yourself so here's how:
- Open the chart txt file you want to edit
- Look for any special characters and copy them to another file and remember where they go (everything below "[TimingPoints]" is just numbers dw about that)
- Temporarily remove the special characters in your chart file and replace them with like idk the letter 'a'
- Run the exe, edit your charts, do whatever you want to use this for
- On the chart output, copy all the special characters back in the same place they originally were
Alternatively just don't add special characters when you create the chart and then after you use this editor you can import the result back into whatever chart editor you used and fix it up there
